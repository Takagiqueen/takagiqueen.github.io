## 起点

我的专业是智能车辆工程。课程里学了 Python、C++，还有 SolidWorks 三维建模。编程一开始只是工具——写点数据处理脚本、跑仿真计算。

## Apollo 比赛

参加 Apollo 自动驾驶比赛之后，头一回感觉到"完整的工程系统"长什么样。调试路径规划和控制算法的时候，我发现我挺喜欢把逻辑写清楚、把边界条件处理干净这件事。

## 从后端摸到前端

从 FastAPI 开始学 Web 后端。做了几个练手项目之后，想做点完整的——于是有了 LeafVault。要做完整应用，前端就绕不开了。原生 JavaScript、浏览器 API（IndexedDB、Service Worker）、响应式布局、PWA，一个一个啃。

## 部署和安全

学 Docker 是因为想让项目在哪台机器上都能跑。学 CSP、CSRF、XSS 是因为做了登录功能之后，安全问题自然就跟过来了。学 pytest 是因为手动测试已经跟不上改代码的速度。

## 现在

能一个人搞定从后端 API、数据库、前端到部署的完整流程。接下来想深入 TypeScript，继续把代码质量和测试补扎实。

## 几个体会

- 别等"学完"了再动手做。项目本身就是最好的学习路径。
- 遇到安全问题别绕，花时间把它搞懂。这些东西面试和实际工作都会用上。
- 车辆工程练出来的系统思维，做 Web 全栈时确实用得上。不是硬扯，是真实感受。
